MDX te permite escribir JSX directamente en tus archivos Markdown. Esto significa que puedes importar componentes, usar variables y crear contenido interactivo — todo dentro de un post de blog.

## Por que MDX?

El Markdown tradicional es excelente para contenido estatico, pero a veces necesitas mas. MDX conecta la simplicidad del Markdown con todo el poder de los frameworks de componentes:

- **Componentes reutilizables** embebidos directamente en el contenido
- **Datos dinamicos** renderizados en tiempo de build
- **Elementos interactivos** que mejoran la experiencia de lectura
- **Contenido con tipos seguros** con soporte completo del IDE

## Embebiendo Componentes

Una de las fortalezas de MDX es la capacidad de embeber componentes directamente en tu escritura. Esto es especialmente util para documentacion tecnica donde quieres mostrar ejemplos interactivos junto a las explicaciones.

export const Aviso = ({ tipo = 'info', children }) => (
  <div style={{
    padding: '1rem 1.5rem',
    borderRadius: '0.5rem',
    marginBottom: '1.5rem',
    backgroundColor: tipo === 'advertencia' ? '#fef3c7' : tipo === 'error' ? '#fee2e2' : '#dbeafe',
    borderLeft: `4px solid ${tipo === 'advertencia' ? '#f59e0b' : tipo === 'error' ? '#ef4444' : '#3b82f6'}`,
    color: '#1e293b'
  }}>
    <strong>{tipo === 'advertencia' ? 'Advertencia' : tipo === 'error' ? 'Error' : 'Info'}:</strong> {children}
  </div>
);

<Aviso tipo="info">
  Este componente de aviso esta definido y renderizado directamente dentro del archivo MDX. No se necesita un archivo de componente separado!
</Aviso>

<Aviso tipo="advertencia">
  Los componentes MDX se renderizan en tiempo de build en el modo de salida estatica de Astro. Para interactividad del lado del cliente, necesitaras usar componentes de framework con directivas client.
</Aviso>

## Usando Variables y Expresiones

MDX soporta expresiones JavaScript en linea. Puedes calcular valores y renderizarlos directamente:

export const estadisticas = {
  lenguajes: 12,
  frameworks: 8,
  herramientas: 25,
  anio: new Date().getFullYear()
};

Aqui hay algunas estadisticas para {estadisticas.anio}:

- **Lenguajes soportados:** {estadisticas.lenguajes}
- **Integraciones de framework:** {estadisticas.frameworks}
- **Herramientas de desarrollo:** {estadisticas.herramientas}
- **Tamano total del ecosistema:** {estadisticas.lenguajes + estadisticas.frameworks + estadisticas.herramientas} tecnologias

## Codigo con Contexto

MDX hace natural intercalar explicaciones con codigo. Aqui tienes una ruta API paso a paso:

**Paso 1:** Definir el manejador de ruta

```typescript
import type { APIRoute } from 'astro';

export const GET: APIRoute = async ({ params, request }) => {
  const { id } = params;
  // Obtener datos basados en el parametro de ruta
  const data = await fetchItem(id);

  return new Response(JSON.stringify(data), {
    status: 200,
    headers: { 'Content-Type': 'application/json' },
  });
};
```

**Paso 2:** Agregar manejo de errores

```typescript
export const GET: APIRoute = async ({ params }) => {
  try {
    const data = await fetchItem(params.id);
    if (!data) {
      return new Response('No Encontrado', { status: 404 });
    }
    return new Response(JSON.stringify(data), {
      headers: { 'Content-Type': 'application/json' },
    });
  } catch (error) {
    return new Response('Error Interno del Servidor', { status: 500 });
  }
};
```

## Cuando Usar MDX

MDX es excelente para:

1. **Tutoriales tecnicos** que se benefician de ejemplos interactivos
2. **Documentacion** con patrones de componentes reutilizables
3. **Contenido basado en datos** donde los valores cambian con el tiempo
4. **Posts con layouts personalizados** que van mas alla del Markdown estandar

Para posts de blog simples con mayormente texto e imagenes, el Markdown estandar (`.md`) es perfectamente adecuado y tiene menos sobrecarga. Elige la herramienta correcta para el contenido.
