# /lib-upgrade — provided by the `deepworkplan` dependency-upgrade addon

Bring this repository's dependencies up to date **without breaking the build**.
This command is a **thin delegator** to the DeepWorkPlan
`deepworkplan-addon-dependency-upgrade` addon — it does not contain the upgrade
logic itself. (This replaces the repo's previous in-tree library-upgrade procedure.)

## Steps

1. Invoke the dependency-upgrade addon: read
   `.agents/skills/deepworkplan/addons/dependency-upgrade/SKILL.md` and follow
   its flow.
2. Follow the addon's flow for this repo:
   - **Package manager:** pnpm (`pnpm-lock.yaml`), with `pnpm dlx npm-check-updates`
     (ncu) for discovery.
   - Classify upgrades (patch / minor / major), upgrade in safe batches.
   - Run this repo's **real** validation gate after each batch:
     `pnpm run biome:check && pnpm run astro:check && pnpm run build`
     (add `pnpm run test` when the change touches `src/lib/`).
   - Revert any batch whose gate fails; summarize at the end.
3. Surface the upgrade report and the diff; do **not** auto-commit.

## Notes

- Majors require explicit approval before they are included in a batch.
- Pin exact versions — this repo forbids caret (`^`) / tilde (`~`) ranges in
  `package.json`.
- The lockfile is regenerated by pnpm — never hand-edit it.
